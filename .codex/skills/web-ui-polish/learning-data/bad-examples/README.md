# Bad Examples

This directory contains screenshots of UI implementations that violate AetherPane design principles.

## Naming Convention
`[issue-type].png`

Examples:
- `too-busy.png` - Visual overload, too many competing elements
- `poor-hierarchy.png` - No clear visual hierarchy
- `weak-glass.png` - Glass effects applied incorrectly or too heavily
- `cramped-spacing.png` - Insufficient breathing room
- `too-many-ctas.png` - Multiple competing call-to-action buttons

## Anti-Pattern Criteria
- Visual Hierarchy Score: < 5
- Breathing Space Score: < 5
- Glass Quality Score: < 5
- Overall Design Score: < 5

## How to Add
1. Find or create an example of a design anti-pattern
2. Take a screenshot (1920x1080 recommended)
3. Name it according to convention
4. Place it in this directory
